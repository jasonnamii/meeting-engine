# Group Pathology — 집단 병리 스크리닝 가이드

회의의 실패는 대부분 "집단의 심리병리"에서 비롯됨. 좋은 퍼실리테이션 기술로도 못 막는 것들이 있다. 따라서 설계 **전**에 반드시 스크린.

---

## 1. Groupthink (Irving Janis, 1972)

**정의**: 응집력 높은 그룹에서 비판적 사고가 억제되고 비합리적 의사결정으로 귀결되는 현상.

**8가지 증상** (모두 나타나지는 않지만 2개 이상 있으면 주의):

| # | 증상 | 신호 |
|---|------|------|
| 1 | Illusion of Invulnerability | "우리는 실수할 리 없다" "우리 전략은 완벽하다" |
| 2 | Collective Rationalization | 반대 증거를 무시하거나 합리화. "그건 다른 상황이다." |
| 3 | Stereotyped Views of Enemy | 경쟁사/반대편을 "무능하거나 악하다"고 단순화 |
| 4 | Direct Pressure on Dissenters | 의견 다른 사람에게 "넌 팀플레이가 없네" 압박 |
| 5 | Self-appointed Mindguards | 일부가 부정적 정보를 자발적으로 걸러냄 (주장: 팀 화합 위해) |
| 6 | Illusion of Unanimity | 침묵을 동의로 착각. "아무도 반대 안 하니까 모두 찬성이지" |
| 7 | Unquestioned Belief in Morality | "우리가 하는 것이 도덕적으로 맞다" → 수단을 정당화 |
| 8 | Pressure on Waveters | 마지막 순간에 의견 바꾼 사람을 강하게 설득해서 원래 의견으로 돌리려 함 |

**구조적 선행조건 (원인)** (세 가지 모두 있을 때 위험):

1. **높은 응집력** (좋은 팀이 오히려 위험): 리더 주변에 최고 성과자들만 모임
2. **외부 압력 + 시간 제약**: 빠르게 결정해야 함 ("3개월 안에 결정해야 해")
3. **독재적 리더십**: 리더가 사전에 선호도를 명시했거나 강하게 주장

**역사적 사례**:
- 베이 오브 픽스(1961): CIA가 "쿠바 침공은 성공할 것"이라고 과신 → 참담한 실패
- 스페이스셔틀 챌린저호(1986): 엔지니어들의 경고 무시하고 발사 강행 → 폭발
- 2008 금융위기: 리만브라더스 리더십이 "우리는 망할 수 없다" 과신

**설계 시 방어 구조**:

| 방어 메커니즘 | 구체적 방법 |
|------------|----------|
| **Devil's Advocate 역할** | 1명을 정해서 "이 결정의 문제점을 찾는 게 너의 일"이라고 명시 |
| **외부 전문가 초청** | 회의의 15~20%를 "조직 밖 사람의 의견"에 할애 |
| **익명 아이디어 수집** | 사전에 구글폼/익명 채팅으로 의견 수집 → 회의에서 공개 |
| **2단계 의사결정** | 첫 회의에서 결정 X → 며칠 뒤 "다시 생각해보니" 단계 마련 |
| **소그룹 독립 검토** | 팀을 둘로 나눠서 각각 반대 의견 쓰기 |
| **신뢰 기반 심문** | "이 의견에 여전히 동의하나? 1%라도 의심스러운 부분?" |

**체크리스트** (설계 전 확인):
- [ ] 이 팀의 응집력이 높은 편인가? (YES → 위험)
- [ ] 리더가 사전에 선호 입장을 명시했는가? (YES → 위험)
- [ ] 외부 압력/시간 제약이 강한가? (YES → 위험)
- [ ] 위 3가지 모두 YES면 → 방어 구조 3개 이상 필수

## 2. Abilene Paradox (Jerry B. Harvey, 1974)

**정의**: "아무도 가고 싶지 않은데 모두가 동의해서 실행한다"는 역설.

**메커니즘**:

```
개인의 진정한 의도: "이건 나쁜 아이디어야"
개인의 추정: "리더는 이걸 원하겠지?"
침묵: "반대하면 팀 문제 일으킬 텐데..."
집단 의도 추정: "모두가 찬성하는 것 같네"
결과: 아무도 원하지 않는 결정이 만장일치로 시행
```

**Groupthink와의 차이**:
- Groupthink: 리더가 명시적으로 강압 (active)
- Abilene Paradox: 침묵과 추정으로 인한 자동화 (passive)

**신호**:
- 회의 후 "혼자 생각하기에는 이상한데" 하는 사람이 많음
- "나만 안 하려고 하는 건 줄 알았는데..."
- 의사결정 직후 급속도로 반발

**설계 시 방어 구조**:

| 방어 메커니즘 | 구체적 방법 |
|------------|----------|
| **사전 익명 폴링** | "이 안에 동의하나요? (1:완전동의 ~ 5:강한거부)" 익명 투표 먼저 |
| **명시적 "나는 이렇게 생각" 선언** | "앞으로 모두 한 번씩, ' 내 의견은...'이라고 말하세요. 침묵 금지." |
| **"진짜 동의인가?" 확인** | 결정 후 1:1 follow-up으로 "너는 진짜 동의하나?" 비공개 확인 |
| **역순 발언** | 가장 아래 직급이 먼저 말함 (리더 마지막) → 리더의 영향 최소화 |
| **비공개 투표 후 공개** | 먼저 비공개 투표로 진짜 의견 파악, 그 후 "사실은 X가 더 나을 것 같은데?" 공개 논의 |

**체크리스트**:
- [ ] 참석자 중 "조용히 있는 사람"이 있는가? (YES → 위험)
- [ ] 리더의 성격이 "강압적이지는 않지만 영향력 강함"인가? (YES → 위험)
- [ ] 조직 문화가 "반대하기 어려운" 분위기인가? (YES → 위험)
- [ ] 위 중 2개 이상 YES → 익명 폴링 + 명시적 의견 선언 필수

## 3. Production Blocking (Diehl & Stroebe, 1987)

**정의**: 브레인스토밍에서 한 명이 말할 때 다른 사람이 생각하지 못함 → 아이디어 20~30% 손실.

**메커니즘**:
- A가 말하는 동안 B는 자신의 아이디어를 미룸
- 시간 제약으로 모두가 말할 수 없음
- 더 말이 많은 사람의 아이디어만 쌓임

**신호**:
- 10명이 모인 1시간 브레인스토밍 → 실제로는 2~3명만 많이 말함
- "아이디어는 많은데 시간이 없다"
- 후에 "그때 말하지 못한 아이디어가 있었어"

**손실 규모**:
- 개인 브레인스토밍(Nominal Group): 8명이 각각 30분 → 아이디어 70개
- 대면 브레인스토밍: 8명이 함께 30분 → 아이디어 37개 (절반 이상 손실)

**설계 시 방어 구조**:

| 방어 메커니즘 | 구체적 방법 | 효과 |
|------------|----------|------|
| **Nominal Group Technique** | 1단계: 각자 5분 조용히 쓰기 → 2단계: 공유 | +60% 아이디어 |
| **Electronic Brainstorming** | 노트북/타블렛에 동시 입력 → 실시간 공유 | +80% 아이디어 |
| **Async + Sync 조합** | 사전 구글폼 입력 → 회의에서 공유+논의 | +40% 아이디어 + 높은 몰입 |
| **라운드로빈** | 한 명씩 아이디어 1개씩만 말함 (같은 시간에 더 많은 사람 발언) | +30% 참여도 |
| **소그룹 분리** | 4명씩 3개 그룹 → 각각 브레인스토밍 → 모음 | +각 그룹에서 Production Blocking 최소화 |

**가장 효과적인 조합**:
1. 사전: 개인별로 아이디어 5개 쓰기 (사전 공유)
2. 회의: Nominal Group Technique (라운드로빈) + 실시간 기록
3. 후속: 온라인으로 추가 아이디어 1주일 동안 계속 받기

**체크리스트**:
- [ ] 이번 회의가 "브레인스토밍"인가? (YES → Production Blocking 위험)
- [ ] 10명 이상인가? (YES → 위험도 높음)
- [ ] 시간이 1시간 이하인가? (YES → 손실 심각)
- [ ] 위 모두 YES → 반드시 Nominal Group 사용

## 4. Social Loafing (Bibb Latané, 1979)

**정의**: 그룹에 속하면 개인 노력이 감소. "누군가는 할 거겠지"

**메커니즘**:
- 그룹이 커질수록 개인의 책임감 ↓
- 개인의 기여가 보이지 않으면 노력 ↓
- 규모: 2명(100% 노력) → 4명(80%) → 8명(60%) → 16명(40%)

**신호**:
- 팀 회의 후 "누가 실행해야 하는데..." 애매함
- 지난 회의 액션 "진전 없음" 많음
- 정기 회의 참석은 하는데 준비는 안 해옴

**설계 시 방어 구조**:

| 방어 메커니즘 | 구체적 방법 | 효과 |
|------------|----------|------|
| **Individual Accountability** | "A가 이것 담당. A가 못 하면 A가 보고합니다" 명시 | +35% 노력 |
| **Visible Contribution** | 각자의 기여를 주간 회의에서 공개 | +40% 개인책임 |
| **작은 그룹** | 의사결정: 3~5명 | +명확한 책임 |
| **실시간 진행도 추적** | "이 항목은 A가 담당하고, 다음 회의에 보고" | +30% 완료율 |
| **개인 성과 기여도 점수** | OKR 달성도 기반 개인별 스코어 | +조직 전체 동기 |

**가장 효과적**: Individual Accountability + 작은 그룹 + Visible Contribution (셋 다 함께)

**체크리스트**:
- [ ] 액션아이템의 담당자가 명확하게 정해지는가? (NO → 위험)
- [ ] 담당자가 "내 일"이라고 느끼는가? (NO → 위험)
- [ ] 다음 회의에서 "누가 뭘 했는지" 공개 보고하는가? (NO → 위험)
- [ ] 위 중 2개 이상 NO → 반드시 accountability 구조 설계

## 5. Anchoring Bias in Groups

**정의**: 첫 정보가 판단을 지배. "첫 제시 숫자에 사람들의 판단이 고정됨"

**메커니즘**:
- 첫 선택지 제시 → 뇌가 그걸 "기준점"으로 설정
- 이후 모든 판단이 그 점으로부터의 상대적 거리로 계산됨

**예시**:
- 신제품 가격 결정: "업계 평균 $99라고 들었는데, 우리는 $120?"
- 첫 제시된 $99 → 뇌가 고정
- 실제로는 $80이 맞아도 "비싸다"고 느껴짐

**신호**:
- 회의에서 "역시 [첫 제시 옵션]이 맞는 것 같아"
- 이후 선택지들이 첫 번째와의 비교로만 논의됨
- 진정으로 최적인 선택보다 "첫 제시"에 수렴

**설계 시 방어 구조**:

| 방어 메커니즘 | 구체적 방법 |
|------------|----------|
| **다중 앵커 동시 제시** | "세 가지 옵션을 동시에 보여줄게요" (1~2초 안에) |
| **역순 제시** | 최고 가격부터 시작 (낮은 가격이 상대적으로 저렴해 보임) |
| **Explicit De-anchoring** | "업계 평균은 $99지만, 그걸 무시하고 최적값을 생각해봅시다" |
| **독립 판단 후 집계** | 모두 먼저 개인적으로 숫자를 쓰게 함 (첫 제시 효과 약화) |
| **기준 재정의** | "시장 조사 기준이 아니라 고객 가치 기준으로" |

**체크리스트**:
- [ ] 논의 시작 전 첫 번째 옵션이 이미 제시되었는가? (YES → 위험)
- [ ] 그 첫 옵션에 대해 "역시", "대체로" 같은 수용적 표현이 나왔는가? (YES → Anchoring 작동 중)
- [ ] 위 둘 다 YES → 반드시 동시 제시 또는 역순 제시 재설계

## 6. Status Effects (Karpowitz & Mendelberg, 2012)

**정의**: 높은 지위/직급의 의견이 더 영향력 있음. "상사 의견 = 옳은 의견"

**메커니즘**:
- 고지위자(리더/전문가)가 더 많이 말함 (70% vs 30%)
- 고지위자의 의견이 더 강하게 수용됨
- 저지위자(신입/비전문가)의 좋은 아이디어도 무시됨

**신호**:
- 리더가 의견 내면 자동으로 모두 그 방향으로 수렴
- "그건 리더님 말이니까 맞겠지"
- 신입이나 직급 낮은 사람의 의견이 잘 안 들려짐

**신경과학적 근거** (fMRI 연구):
- 고지위자의 얼굴을 보면 뇌의 이성 영역 불활성화 → 비판적 생각 ↓

**설계 시 방어 구조**:

| 방어 메커니즘 | 구체적 방법 | 효과 |
|------------|----------|------|
| **역순 발언순서** | 가장 아래 직급이 먼저 말함, 리더는 마지막 | +신입/직급 낮은 사람의 발언 시간 200% ↑ |
| **Peer-to-peer 먼저** | 리더를 빼고 팀원끼리 먼저 논의 | +다양한 의견 |
| **Written-first 프로토콜** | 말 전에 먼저 의견을 종이에 씀 | +리더의 사전 영향 최소화 |
| **익명 의견 수집** | 구글폼으로 익명 투표 → 결과 공개 | +저지위자의 진정한 의견 파악 |
| **"리더는 판단만, 의견 안 함" 명시** | "내(리더)는 의견 안 하고 판단만 합니다" | +팀원들의 자유로운 발언 |

**가장 효과적**: Written-first + 역순 발언 (둘 다)

**체크리스트**:
- [ ] 리더가 회의 중에 자신의 의견을 명시했는가? (YES → 위험)
- [ ] 이후 팀원들의 의견이 리더 의견에 집중되는가? (YES → Status Effects 작동)
- [ ] 신입/직급 낮은 사람이 발언 시간 10% 이하인가? (YES → 위험)
- [ ] 위 중 2개 이상 YES → 역순 발언 + written-first 필수 설계

## 7. Noise in Judgment (Daniel Kahneman, 2021)

**정의**: 같은 정보를 받아도 판단이 다름. Bias(체계적 오류)가 아니라 Noise(무작위 변동).

**메커니즘**:
- A와 B가 같은 정보를 받음
- A: "우리는 이 방향으로" (판단 1)
- B: "아니 보니까 저 방향이 맞아" (판단 2)
- 누가 맞는지가 아니라, **둘 다의 편차가 큼** = Noise

**소스**:
- 개인적 기분/피로
- 인지적 편향 차이
- 경험/스킬 차이
- 취향 차이
- 회의 시간대 (아침 vs 오후)

**신호**:
- 같은 회의에서 "의견이 제각각"
- 지난번과 다른 판단 (조건은 같은데)
- 10명의 의사결정 점수가 정상분포를 이룸

**설계 시 방어 구조 (Decision Hygiene Checklist)**:

| 단계 | 목적 | 구체적 방법 |
|------|------|-----------|
| **①독립 판단** | 타인 영향 제거 | 모두가 먼저 개인적으로 점수/의견을 씀 (10분) |
| **②집계** | 분산 파악 | 점수들을 모으고 "얼마나 차이나나?" 확인 |
| **③근거 소통** | 왜 그렇게 판단했나 | 극단값(가장 높음/낮음)을 낸 사람이 근거 설명 |
| **④재판단** | 집단 지혜 | 근거를 들은 후 다시 판단 (대부분 수렴됨) |
| **⑤최종 결정** | 오너십 | 리더가 최종 판단 + 근거 기록 |

**핵심**: ①③을 꼭 거쳐야 함. 안 그러면 가장 말이 많은 사람의 판단으로 수렴.

**체크리스트**:
- [ ] 의사결정 시 모두가 먼저 독립적으로 판단했는가? (NO → Noise 크다)
- [ ] 판단들의 분산을 확인했는가? (NO → 왜 차이나는지 모른 채 진행)
- [ ] 극단값을 낸 사람의 근거를 들었는가? (NO → 소수의견 무시)
- [ ] 위 모두 NO → Decision Hygiene 프로토콜 도입 필수

## 전체 스크리닝 프로토콜 (설계 전 5분)

**질문을 순서대로 물어보면서 체크하기**:

| 병리 | 스크리닝 질문 | YES면 위험도 |
|------|------------|----------|
| **Groupthink** | 1. 이 팀의 응집력이 높은 편인가? 2. 리더가 선호도를 명시했는가? 3. 외부 압력/시간 제약이 강한가? | 3개 모두 YES → 고위험 |
| **Abilene Paradox** | 1. 조용한 사람이 있는가? 2. 반대하기 어려운 분위기인가? 3. 첫 발언 후 대부분 찬성으로 수렴하는가? | 2개 이상 YES → 중위험 |
| **Production Blocking** | 1. 이 회의가 브레인스토밍인가? 2. 10명 이상인가? 3. 1시간 이하인가? | 3개 모두 YES → 고위험 |
| **Social Loafing** | 1. 액션 담당자가 명확한가? 2. 참석자가 모두 "내 일"이라고 느끼는가? 3. 다음 회의에 진척도 보고하는가? | 2개 이상 NO → 중위험 |
| **Anchoring** | 1. 회의 전에 첫 옵션이 제시되었는가? 2. 그에 대해 수용적 의견이 나왔는가? | 둘 다 YES → 중위험 |
| **Status Effects** | 1. 리더가 의견을 명시했는가? 2. 이후 팀원이 그 방향으로 수렴하는가? 3. 신입/저직급의 발언 시간이 적은가? | 2개 이상 YES → 중위험 |
| **Noise** | 1. 팀원들의 판단이 제각각인가? 2. 사전에 독립 판단을 수집했는가? | 1번 YES, 2번 NO → 중위험 |

**위험도 종합**:
- 고위험 2개 이상 → 설계 전 방어 구조 3개 이상 필수
- 중위험 3개 이상 → 설계 전 방어 구조 2개 이상 필수
- 저위험만 → 표준 설계로 충분

**위험 병리별 방어 구조 우선순위**:

1. **Groupthink**: Devil's Advocate + 익명 의견 수집
2. **Abilene Paradox**: 익명 폴링 + 명시적 의견 선언
3. **Production Blocking**: Nominal Group Technique
4. **Social Loafing**: Individual Accountability + Visible Contribution
5. **Anchoring**: 다중 앵커 동시 제시 또는 역순 제시
6. **Status Effects**: 역순 발언 + Written-first
7. **Noise**: Decision Hygiene (①독립판단 ②집계 ③근거 소통 ④재판단)
